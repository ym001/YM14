PRESENTATION

YM14 est un protocole de cryptage polymorphe.

Le message est crypté de façon polymorphique ce qui signifie que pour le même message clair le message crypté peut prendre différente forme. Dans le cas du protocole YM14 il y a une quasi infinité de forme pour le message crypté. Ce qui lui confère une résistance importante face à la cryptanalyse. L'entropie du méssage complet est maximale.

UTILISATION

Vous pouvez écrire le message à crypter dans le fichier "message.ym14" ou, si ce fichier n'est pas dans le répertoire de travail de l'émission il vous sera demandé de saisir un message.

Pour émettre un message crypté, vous devez avoir le fichier YM14Emission avec les droit d'execution et permettre l'écriture sur le fichier messageCrypte.ym14.

Pour décrypter un message, vous devez avoir le fichier YM14Reception et le fichier messageCrypte.ym14 dans un même répertoire.

Les caractères autorisés pour l'écriture du message sont: " abcdefghijklmnopqrstuvwxyz0123456789.,?-ABCDEFGHIJKLMNOPQRSTUVWXYZéèà"

Pour cette version de test accessible sur github, il y a uniquement trois utilisateurs bob, alice et anonymous.

Les pass phrases de ces trois utilisateurs sont
  • pour alice "jesuisuneblonde"
  • pour bob "jesuisungeek"
  • pour anonymous ""

Ce protocole est en back tout back. Les pass phrases sont là pour éviter toute attaque par dictionnaire (elle ne sont pas limitées en taille). L'annuaire est une autorité de référence permettant la vérification des identités émettrice et réceptrice. Il est composé des condensats de chaque utilisateur. L'annuaire en vérifiant les identités bloque toute tentative d'attaque du type man in the middle.

Vous avez deux possibilités :
  • soit vous placez l'annuaire dans le dossier d'utilisation au quel cas vous n'avez pas besoin de connexion internet
  • soit vous laissez l'accés à l'url suivant http://ym001.github.io/YM14/annuaire.ym14 pour une lecture en ligne de l'annuaire.

Les fichiers sont compilés pour une architecture linux 64 bits type ubuntu/débian.

Vous pouvez maintenant avoir des communications sécures avec vos interlocuteurs.
